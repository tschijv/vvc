  • Gebruik-raadpleger → ✅ GEMEENTE_RAADPLEGER

3. Autorisatie

  • ✅ Gemeenten zien elkaars landschappen
  • ✅ Leveranciers zien alleen eigen aanbod
  • ✅ Bezoekers zien openbare info

4. Suite-concept

Voorlopig losgelaten. Geen actie nodig.

Conclusies

✅ Sterke punten

  1. Kernfunctionaliteit — Pakketten, leveranciers, gemeenten, standaarden volledig
  2. GEMMA-integratie — Views, glossary, AMEFF, Linked Data, DCAT
  3. Autorisatiemodel — 10 rollen, correct filteren
  4. Zelf-registratie — Publieke registratie + goedkeuring + e-mail
  5. AI-adviseur — Claude-aangedreven advies
  6. Vergelijkfuncties — Jaccard, bulk, side-by-side
  7. API — REST + OpenAPI + RDF + RSS
  8. Deploy & onderhoud — Eén-klik, prompt, PvE-analyse
  9. Performance — Indexes, lazy-loading, select, WebP
  10. 24 extra features — Buiten PvE om gebouwd

❌ Resterende gaps

  1. 2FA/TOTP — Alleen email + wachtwoord
  2. Multi-organisatie — 1 user = 1 org
  3. OTAP — Niet geconfigureerd
  4. IBD/compliance — Volledig open terrein
IBD-wensen: volledig open terrein
Alle IBD-gerelateerde wensen (BIO, DPIA, pen-tests, DigiD, kwetsbaarheden, AVG) zijn niet gerealiseerd. Groot domein, deels afhankelijk van GEMMA-uitbreidingen.

Aanbeveling prioritering

  1. Beveiligingseisen — 2FA/TOTP implementeren
  2. Gebruikersbeheer — multi-organisatie toegang
  3. DevOps — OTAP omgevingen configureren
  4. IBD/compliance — volledig nieuw te bouwen

Prioritering bij harde eisen (2FA, OTAP) — randvoorwaardelijk voor productie-gebruik.
